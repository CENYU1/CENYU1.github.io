Git：版本号

发表于2023-03-10|Git
|阅读量:

文末的图片是精华哦！

Git版本号

Git 会根据我们提交的内容通过 SHA-1 加密算法计算得出一个长度为 40 位的 16 进制的版本号。

这个40位的版本号可以在分支合并的时候避免版本的冲突，也可以定位仓库中的文件：2(文件夹) + 38(文件名)

举例

首先

我们添加了一个文件，进行 commit 后产生的版本号是 a0dd3557ae2c89c3f78fbb923d5c061406652e2e。我们可以通过前两位 a0 来定位文件所在的文件夹 ./.git/objects/a0/dd3557ae2c89c3f78fbb923d5c061406652e2e。

我们想要查看这个文件到底是什么内容，但是我们无法直接打开。通过 git cat-file -p a0dd3557ae2c89c3f78fbb923d5c061406652e2e 进行查看。此时，输出的内容是当前的 提交信息。

tree 后面的内容也是一个版本号，进行查看。此时，输出的内容是是文件的 状态信息，其中 100 代表这是一个普通文件，644 是文件的权限，blob 代表当前是一个普通的文件块对象，后面也是一个版本号

对新输出的版本号进行查看。输出的是 文件内容

然后

再添加一个新的文件，版本号是 c2c2c09513338a624f3ef0f2caf33148e4c7057a，进行查看。依然是我们的 提交信息。此时出现两个版本号，其中 parent 的版本号是我们上一次提交的版本号。

对 tree 后的版本号进行查看。此时输出的状态信息关联着两个文件，分别是 .gitattributes 和 a.txt

再然后

对刚才新添加的文件进行修改，版本号是 c9156411db5da624f1a34bb5903daff0837042db，进行查看。此时出现的 parent 的版本号是我们上一次提交的版本号。

对 tree 后的版本号进行查看。此时输出的状态信息 .gitattributes 仍然是第一次提交的 .gitattributes，但 a.txt 已经是新版本的 a.txt。

最后

删除上一步修改的文件，版本号是 6019cf795eb499bd62039fe92165faf6e318ff96，进行查看。此时出现的 parent 的版本号是我们上一次提交的版本号。

对 tree 后的版本号进行查看。此时只出现了 .gitattributes，从这里我们可以看出来，我们的删除操作就是文件状态不指向已经删除了的文件了，也就是说我们没有把文件真正地删除，文件依然保存在我们的仓库中，只不过这个文件是我们历史操作当中的文件，在我们最新的操作当中并没有这份文件，因为被删除了的文件没有被关联上。

Git 是怎么知道最新的版本是哪一个呢

./.git/HEAD 文件中的内容是 ref: refs/heads/main，查看 ./.git/refs/heads/main 文件中的内容是 6019cf795eb499bd62039fe92165faf6e318ff96 与我们最后一次的提交信息的版本号是相同的。
