React 状态的更新是异步的

先看下面的 Demo

为什么调用了 this.setState(…) 后, 页面中 count 的值变为了 1, 但 log 出来的是 0 呢?

setState 本身是同步的方法, 我们在主线程亲自调用, 但是 setState 引起 React 后续更新状态的动作是异步的。相当于 React 已经知道自己需要去修改数据了, 但不是立刻就去做, 代码先往下执行。

如何修改呢？

通过给 setState() 方法传入第二个参数, 即可选的回调函数. 该回调函数在状态更新完毕、界面也更新后(render调用后)才被调用.

关于 setState 第一个参数的另一种写法

第一个参数为一个返回状态改变对象对象的函数, 该函数可以接收到 state 和 props 两个参数
