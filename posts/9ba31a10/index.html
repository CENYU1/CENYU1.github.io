Git：一些操作

发表于2023-03-20|Git
|阅读量:
  • git status --oneline 简单查看 status
  • 文件误删除恢复
    • 删除前已经进行过 commit 操作，现在立马想恢复 git restore a.txt
    • 删除后又进行了 commit 操作，此时最新的提交已经没有 a.txt 了，使用 git reset --hard 之前的版本，但是会丢失提交的过程
    • 删除后又进行了 commit 操作，使用 git revert 需要恢复版本的下一个版本 ，此时又创建了一次新的提交
  • 创建分支 git branch dev
  • 查看有哪些分支 git branch -v
  • 切换分支 git checkout dev
  • 创建分支并切换 git checkout -b order
  • 删除分支 git branch -d order
  • 在 master 分支合并 dev 分支 git merge dev。遇到冲突的话，人为进行代码的修改，再 git add 冲突的文件，git commit -m xxx
  • 查看标签 git tag
  • 给某一次提交添加标签(别名) git tag 标签名 版本号
  • 删除标签 git tag -d 标签名
  • 查看某一次提交及之前的提交 git log 版本号/标签名
  • 利用标签完成分支的创建 git checkout -b 标签名
